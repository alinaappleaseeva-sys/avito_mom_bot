# Avito Mom Bot 🧸

Telegram-бот — умный помощник для занятых родителей, который упрощает процесс продажи и отдачи детских вещей на Авито. 

## Особенности и функционал
- 📝 **Сбор информации:** Бот в формате интерактивного диалога собирает все необходимые детали о детской вещи (категория, состояние, бренд).
- 🏷 **Генерация объявлений:** Автоматически создает честные, но привлекательные тексты для объявлений вашей площадки.
- 💰 **Оценка стоимости:** Примерная оценка оптимальной цены на основе введенных характеристик вещи, а также прогнозирование вероятных сроков продажи.
- 📸 **Чек-листы для фото:** Подсказки, как правильно сфотографировать конкретную вещь, чтобы выделить ее на фоне остальных.
- 📊 **Отслеживание объявлений:** Возможность сохранять ссылки на опубликованные объявления и отслеживать их текущий статус.

## Архитектура решения

Проект построен по слоистой (layered) архитектуре, что позволяет легко заменить временные моки (MVP) на реальные интеграции с Авито в будущем.

```mermaid
flowchart TD
    %% Пользователь и Telegram
    User(("Пользователь")) <--> TG["Telegram API"]
    TG <--> Bot["Bot Dispatcher<br>(aiogram)"]
    
    %% Хендлеры (Контроллеры)
    subgraph Handlers
        add_item[add_item.py]
        active_items[active_items.py]
        errors[errors.py]
    end
    Bot --> Handlers
    
    %% Сервисы (Бизнес-логика)
    subgraph Services["Services (Mock MVP)"]
        pe[price_estimator.py]
        tg[text_generator.py]
        pc[photo_checklist.py]
    end
    add_item -->|Передача Enum-типов| Services
    
    %% База данных
    subgraph Database
        crud[crud.py]
        db[(SQLite\navito_bot.db)]
    end
    Handlers --> crud
    crud --> db
```

1. **Telegram API & Handlers:** Обрабатывают входящие сообщения и стейт-машину (FSM). Они валидируют ввод и преобразуют пользовательский текст в строгие системные типы (Enums).
2. **Services (Mock MVP):** Чистая бизнес-логика. На данном этапе внутри находятся "заглушки" (вместо нейросетей — эвристики и захардкоженные цены). Сервисы ничего не знают про Телеграм и работают только со своими аргументами в виде нормализованных типов `Enum`, возвращая финальный текст и цены.
3. **Database (CRUD):** Слой абстракции над SQLAlchemy. Хендлеры отправляют в CRUD базовые типы (строки/числа). Значения Enum могут быть сохранены в виде строк, и CRUD управляет логикой записи в SQLite.

## Стек технологий
- **Язык разработки:** Python 3.9+
- **База данных:** SQLite
- **ORM:** SQLAlchemy

## Установка и запуск (для разработчиков)

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/alinaappleaseeva-sys/avito_mom_bot.git
   cd avito_mom_bot
   ```

2. **Создайте и активируйте виртуальное окружение:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # Для macOS/Linux
   # Для Windows: venv\Scripts\activate
   ```

3. **Установите зависимости:**
   Текущий проект использует `pyproject.toml`. Для установки зависимостей (включая dev):
   ```bash
   pip install .[dev]
   ```

4. **Инициализация Базы Данных:**
   Перед первым запуском бота необходимо инициализировать базу данных SQLite. Запустите скрипт:
   ```bash
   python -m database.init_db
   ```
   Если вы хотите начать с "чистой" базы, просто удалите файл `avito_bot.db` и запустите эту же команду снова.

   Скопируйте пример файла настроек и заполните его своими данными.
   ```bash
   cp .env.example .env
   ```
   *Обязательно добавьте туда токен вашего Telegram-бота (его можно получить у [@BotFather](https://t.me/BotFather) в Telegram).*

> **💡 Подсказка по миграциям БД:** Бот вызывает автоматический скрипт миграции SQLite `scripts/migrate_statuses.py` при запуске (`bot.py`). Скрипт получает имя базы из `config.DATABASE_URL`. Если вы переносите базу или меняете схему вручную, следите за наличием столбцов `views`, `contacts` и `last_synced_at`.

5. **Запустите бота:**
   ```bash
   python bot.py
   ```

## 🛠 Avito API Integration & Testing

Бот поддерживает три режима работы интеграции с Avito API. Это позволяет безопасно разрабатывать и отлаживать функционал без риска опубликовать тестовые объявления на «боевом» аккаунте или получить блокировку. Режимом управляет флаг `AVITO_API_MODE` в `.env`.

### 1. `mock` (По умолчанию)
Имитация API. Бот симулирует сетевые запросы с небольшой задержкой (asyncio.sleep) и возвращает случайные/фейковые ID и параметры (например, 15 просмотров). В этом режиме никакие запросы к Авито **не уходят**. Безопасно для разработки интерфейса и логики БД.
*Настройки `.env`:*
```env
AVITO_API_MODE=mock
```

### 2. `sandbox` (Песочница)
Специальный режим для тестирования реального сетевого взаимодействия (например, авторизации OAuth2 или закрытия объявлений). В этом режиме `AvitoClient` отправляет реальные REST-запросы к серверам Avito.
- **Обычные пользователи:** При попытке создать объявление видят заглушку: *"Бот находится на техобслуживании, создание объявлений временно приостановлено"*. Мусор от пользователей не попадает в API.
- **Администратор (`TELEGRAM_ADMIN_ID` в .env):** Имеет доступ к специальным скрытым командам отладки (см. раздел "Debug-команды админа").

*Настройки `.env`:*
```env
AVITO_API_MODE=sandbox
TELEGRAM_ADMIN_ID=123456789     # Ваш Telegram ID
AVITO_CLIENT_ID=your_id
AVITO_CLIENT_SECRET=your_secret
AVITO_USER_ID=your_manager_id
```

### 3. `real` (Боевой режим)
Полная интеграция. Все пользователи бота отправляют свои объявления в боевой кабинет Авито, получают реальную статистику. 
*Внимание:* для автоматической публикации объявлений профилю на Авито требуются права бизнес-аккаунта (модели Autoload/CPA).
```env
AVITO_API_MODE=real
# Обязательны все токены, аналогично sandbox
```

### Отладка: Debug-команды админа
Для проверки интеграции администратору (ID из конфига) доступны следующие команды:
- `/debug_avito_ping` — проверяет получение `access_token` `client_credentials`. Не создает никаких объявлений. Полезна для проверки правильности `CLIENT_ID` и `CLIENT_SECRET`.
- `/debug_avito_test_listing` — создает черновик/объявление в "тихой" категории и **немедленно** отправляет запрос на его снятие с публикации (`archive_listing`). Это убережет ваш аккаунт от жалоб за "тестовый мусор".

> **🔒 Безопасность секретов:** Бот содержит встроенный механизм санирования (`_redact()`), благодаря которому `access_token` и `client_secret` автоматически заменяются на `[REDACTED]` при любом падении логгера или в текстах исключений API бота.

**Важно:** Конструирование JSON (`payload`) для создания объявления происходит через `services/avito_mapper.py`.

### Текущая поддержка маппинга (Avito Mapper)

Бот поддерживает внутреннюю доменную модель, которая переводится в официальные категории Авито:
- Коляски (`stroller`) ➡️ **Детские коляски**
- Одежда (`clothes`) ➡️ **Детская одежда**
- Обувь (`shoes`) ➡️ **Детская обувь**
- Игрушки (`toys`) ➡️ **Игрушки**
- Прочее (`other`) ➡️ **Товары для детей и игрушки / прочее**

Если категория не распознана, используется безопасный fallback: `Товары для детей и игрушки / прочее`. Местоположение (geo-позиция) по умолчанию временно захардкожено как `Москва` (в рамках MVP).

## Структура проекта
* `bot.py` — Точка входа в приложение и запуск.
* `config.py` — Загрузка токенов и конфигураций.
* `handlers/` — Обработчики команд и сообщений Telegram (добавление вещей, просмотр списка и т.д.).
* `services/` — Бизнес-логика (генерация текстов, расчет стоимости, подсказки для фотографий).
* `database/` — Модели базы данных и CRUD-операции.
* `utils/` — Вспомогательные утилиты и текстовые конфигурации.
