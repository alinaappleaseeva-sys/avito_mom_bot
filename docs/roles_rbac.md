# Система Ролей и RBAC (Role-Based Access Control)

Бот поддерживает встроенную систему ролей для разграничения прав доступа. В системе предусмотрены две основные роли: `user` (пользователь) и `admin` (администратор).

## Источник истины
Источником прав доступа (Source of Truth) является поле `role` в таблице `users` базы данных.

**Исключение:** Существует так называемый "System Root" администратор. Его Telegram ID зафиксирован в конфигурационном файле `.env` в переменной `TELEGRAM_ADMIN_ID`. Этот пользователь:
1. Автоматически получает права администратора при проверке `IsAdminFilter()`.
2. При первой регистрации (вызове `get_or_create_user_from_telegram`) его запись в БД автоматически получает `role="admin"`.

## Назначение ролей ("Задним числом")
Так как источником истины является БД, вы можете назначить любого пользователя администратором (или наоборот, понизить его полномочия), изменив его статус:

1. **Вручную через SQLite**:
   ```sql
   UPDATE users SET role = 'admin' WHERE telegram_id = <ID>;
   ```
2. **Через будущие административные команды**: Механизм фильтров гибкий и полагается на БД. Как только поле поменяется на `admin`, `IsAdminFilter()` мгновенно выдаст права пользователю.

## Фильтры aiogram
Проверка роли осуществляется с помощью `utils.filters.IsAdminFilter`. Эта логика гарантирует, что вам не нужно "размазывать" проверку `message.from_user.id == config.TELEGRAM_ADMIN_ID` по всем файлам с хендлерами.

```python
from utils.filters import IsAdminFilter

@router.message(Command("admin_only"), IsAdminFilter())
async def admin_handler(message: Message):
    pass
```
