# Линейный AI-пайплайн (Linear Pipeline Design)

Этот пайплайн является отправной точкой до того, как мы внедрим полноценный цикл рассуждений (агентность). Он проходит детерминированный путь.

## Входные данные
- **Требования к серверу**: 1 vCPU, 2 GB RAM, 30+ GB NVMe/SSD, Ubuntu 22.04/24.04.
- **Требования к биллингу**: Pay-as-you-go или 1-3 месяца без обязательного годового депозита (бюджет: строго до 2000 руб за 3 месяца).
- **Провайдеры**: `["Selectel", "RUVDS", "FirstVDS", "Timeweb"]` (с приоритетом слева направо).

## Шаги пайплайна

### Шаг 0: Сбор требований и подтверждение приоритетов
Краткий предварительный LLM-диалог с пользователем:
- Подтверждает параметры железа и желаемого региона.
- Фиксирует отношение к депозитам (максимальная неприязнь).
- Дает возможность менять приоритеты (например, "если разница в цене < 15%, предпочитай pay-as-you-go").

### Шаг 1: Сбор данных `search_tariffs()`
- Обход списка всех провайдеров. Сбор тарифных сеток.
- Начать можно с загрузки мок-данных (JSON), имитирующих реальные ответы провайдеров для отладки.

### Шаг 2: Анализ биллинга `analyze_billing(offers)`
- **Код + LLM**: Отсев тарифов.
- Оставляются только те, где >=1 vCPU, >=2GB RAM, >=30GB Диск, регион РФ. 
- Жесткий фильтр стоимости (<= 2000 руб за 3 месяца).
- **LLM Промпт**: *"Отфильтруй список: оставь только почасовую оплату (pay-as-you-go) ИЛИ оплату за 1-3 месяца без условия внесения годового депозита. Запрещены табы '700р/мес при оплате за год'."*

### Шаг 3: Сравнение кандидатов `compare_candidates(candidates)`
- Сначала собираются кандидаты со **всех** провайдеров, формируя один общий пул.
- **Код (Грубый скоринг)**: Скрипт прогоняет скоринг по понятной формуле (например: минимальная цена за 3 месяца = баллы, pay-as-you-go = баллы, приоритетный провайдер = баллы) и жестко сортирует кандидатов.
- **LLM (Объяснение)**: Самые топовые кандидаты (или один лучший) передаются в LLM для генерации объяснения, почему этот вариант оптимален по условиям биллинга.

### Шаг 4: Генерация инструкций `generate_checkout_flow(provider, tariff)`
- **Вход:** Выбранный провайдер и конкретный тариф.
- **LLM Промпт:** *"Для тарифа {tariff} от {provider} составь пошаговое руководство покупки..."*
- **Сверка с реальностью (Обязательный запрос фидбека)**: Прямо в финальном Markdown агенту поручено вписать шаг: 
  > *"Сверься с реальным экраном: есть ли где-то слова или галочки вроде 'годовой', 'депозит', 'предоплата за год'. Если есть — остановись и сообщи мне!"* 
  Это уменьшает риск галлюцинаций LLM об идеальном тарифе.

### Шаг 5: Проверка результата `validate_result(vps_info)`
- Сбор фактических данных от юзера после покупки (скриншоты, подтверждение платежа на сайте).
- **LLM** проверяет соответствие.
- **Запись в память**: Если на шаге 4/5 всплывет скрытый депозит, этот конкретный тариф заносится в "базу засад" (память пайплайна/агента), чтобы исключать его в будущих запусках.
