# Дизайн Агента (Agentic Design)

Поверх линейного пайплайна в будущем будет внедрен агентный цикл (ReAct или StateGraph/LangGraph-подобный механизм). Это позволит агенту быть более самостоятельным и гибким, если у провайдеров изменятся интерфейсы или если поиск не даст подходящих результатов.

## Цикл Агента (Observe -> Think -> Act -> Check)

1. **Observe (Наблюдение)**: Агент получает задачу от пользователя (найти VPS для avito_mom_bot) и текущий контекст.
2. **Think (Размышление)**: Агент анализирует ограничения (оплата без годового депозита) и приоритеты (нужно начать с Selectel). Принимает решение вызвать тул сбора тарифов для первого провайдера.
3. **Act (Действие)**: Вызов тула `search_tariffs("Selectel")`, затем `analyze_billing(сборка_Selectel)`.
4. **Check (Проверка / Observe-2)**: Анализ полученных ответов от тулов. 
   - Достаточно ли данных? 
   - Есть ли подвох с депозитом? 
   - Если есть подвох или нужных тарифов нет — агент возвращается на Think, чтобы проверить следующего провайдера (RUVDS).
5. **Finish**: Когда кандидат найден и выбран (`compare_candidates`), агент собирает финальный результат через `generate_checkout_flow` и отдает пользователю. Впоследствии ожидает `vps_info` от юзера для `validate_result`.

## Инструменты агента

### 1. `search_tariffs(provider)`
- **Сигнатура**: `(provider: str) -> List[Dict]`
- **Где вызывается**: На этапе Act, когда агент решает собрать информацию по конкретному провайдеру в порядке приоритета.
- **Безопасность**: Инструмент только собирает (читает) данные из внешнего мира (парсинг сайтов/API, Read-Only). Никаких действий от имени пользователя.

### 2. `analyze_billing(offers)`
- **Сигнатура**: `(offers: List[Dict]) -> List[Dict]`
- **Где вызывается**: Сразу после получения списка тарифов, чтобы отфильтровать "годовые ловушки" и слишком слабые сервера. Может вызывать LLM под капотом для интерпретации "мутных" описаний.
- **Безопасность**: Чисто аналитическая/математическая функция. Безвредна.

### 3. `compare_candidates(candidates)`
- **Сигнатура**: `(candidates: List[Dict]) -> Dict`
- **Где вызывается**: Когда есть несколько подходящих тарифов (возможно, от разных провайдеров), чтобы выбрать лучший по цене/условиям.
- **Безопасность**: Только возвращает оптимальный вариант (dict). Безвредна.

### 4. `generate_checkout_flow(provider, tariff)`
- **Сигнатура**: `(provider: str, tariff: Dict) -> str`
- **Где вызывается**: Перед завершением работы агента (переход в Finish), чтобы сформировать читаемый Markdown-чеклист для пользователя.
- **Ограничения по безопасности**: Инструмент *только формирует текст*. Он **не производит** никаких действий в браузере, **не переходит** к оплатам и **не запрашивает** данные карты у пользователя внутри бота.

### 5. `validate_result(vps_info)`
- **Сигнатура**: `(vps_info: str) -> str`
- **Где вызывается**: Отдельным вызовом (когда пользователь возвращается к агенту и приносит IP, скриншоты или логины после покупки).
- **Ограничения по безопасности**: Агент может подсказать команды по деплою проекта, но сам **не выполняет запуск произвольных скриптов**. Он просто анализирует `vps_info` и дает рекомендации текстом на основе предоставленных данных.
